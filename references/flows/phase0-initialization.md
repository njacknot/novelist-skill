# 第0步：初始化与偏好加载

**每次触发SKILL时首先执行：**

## 1. 读取用户偏好

检查 `user-preferences.json` 是否存在
- 存在 → 加载偏好数据，用于后续选项排序、推荐标记、个性化欢迎语
- 不存在 → 使用默认值，首次交互结束后自动创建

偏好字段最小集：`favoriteGenres`、`preferredProtagonist`、`preferredPerspective`、`preferredTone`、`typicalChapterCount`、`styleReferences`、`dislikes`、`creationHistory`。

## 2. 快捷入口检测

分析用户的首次输入是否已包含丰富的创作信息（题材、主角、冲突等关键要素）

- 如果信息充足 → 展示提取结果，提供三个选项：
  - "是的，直接开始规划" → 跳过问答，进入第二阶段
  - "先展示提取的信息让我确认/修改" → 展示配置供确认后进入规划
  - "不，我还是想走一遍问答流程" → 进入第一层问答
- 如果信息不足 → 直接进入第一层问答

## 3. 个性化欢迎

（如有偏好数据）：

> "欢迎回来！基于你之前的创作习惯，我为你做了一些推荐（标记为⭐的选项）。"

## 4. 中断续写检测

扫描 `./novelist-projects/` 目录下的项目文件夹：

- 查找含 `02-写作计划.json` 且 `status` 为 `"in_progress"` 或 `"validating"` 的项目
- 如果找到未完成项目：
  - 展示项目信息：小说名称、完成进度（X/Y 章已完成）
  - 提供选项：
    - "继续上次创作" → 直接进入第三阶段，从中断章节继续
    - "开始新作品" → 进入第一层问答
- 如果未找到 → 正常进入快捷入口检测或第一层问答

## 5. 共享机制（内联摘要）

### 三大黄金法则

1. 展示而非讲述
2. 冲突驱动剧情
3. 悬念承上启下

### 偏好更新规则

- 每完成一层问答：静默更新 `user-preferences.json`
- 用户说“记住我的偏好”：保存当前配置
- 用户说“忘记XX偏好”：清除指定字段
- 用户说“重置偏好”：清空偏好
- 完成一部长篇：追加 `creationHistory`

### 标题传递机制

- 标题通过对话上下文在 Phase 1 → Phase 2 传递
- Phase 2 将标题落盘到：项目目录名、`02-写作计划.json.novelName`、`01-大纲.md` 文件头

### 写作计划系统（`02-写作计划.json`）

- 记录章节状态（pending/in_progress/completed/failed）
- 记录写作模式（serial/subagent-parallel）
- 作为中断续写与校验依据

### 容错

- 偏好文件损坏：忽略偏好并使用默认值
- 写作计划 JSON 损坏：从大纲摘要 + 章节文件状态尝试重建进度

## 6. 交互式脑洞引导（内联摘要，规划中）

用于“用户想法很模糊”场景的 5 轮收敛：

1. 核心种子提取（题材/冲突/特殊设定/情绪基调）
2. 世界观拓展（规则、社会结构、地理格局）
3. 角色网络构建（主角、核心配角、反派动机）
4. 剧情骨架搭建（三幕与转折、伏笔线）
5. 确认与收敛（输出结构化草案）

若用户“没想好”：提供 2-3 个可选方案并允许直接选择。
