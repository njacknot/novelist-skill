# 质量趋势仪表盘规范

> **版本**：v1.0（v1.2.0 引入）
> **定位**：跨章质量诊断工具——让"风格漂移 / 追读力下滑 / 一致性损耗"等慢病在拐点出现时**立即可见**
> **触发**：`/面板 --质量`（详见 [command-contracts.md](./command-contracts.md) §3.7）
> **脚本**：`scripts/quality_dashboard.py`
> **输入**：`02-写作计划.json` 的 `chapters[].gateScores` 数据
> **输出**：CLI 表格 + ASCII 折线图 / JSON / Markdown 报告（可选）

---

## 为什么需要仪表盘

**当前盲区**：每章独立判定 pass/fail，但**无法回答**：
- 写到第 50 章时质量是不是在掉？
- 文风从第几章开始漂移？
- 读者维度的子分项哪个最弱？哪个在改善？
- 哪几章是"拖累全书均值"的离群章？

竞品对比：
- `EdwardAThomson.NovelWriter` — Quality Analytics + 配置阈值 + 趋势告警
- `webnovel-writer` — Dashboard 显示项目状态 / 实体图谱 / 追读力折线

## 数据基础：02-写作计划.json 的扩展字段

每个 `chapters[i]` 增加以下字段（向后兼容，老项目缺字段时仪表盘自动跳过）：

```jsonc

  "chapterNumber": 1,
  "title": "最后一班列车",
  "filePath": "第01章-最后一班列车.md",
  "status": "completed",
  "wordCount": 3245,
  "wordCountPass": true,
  "retryCount": 0,

  // ★ v1.2.0 新增字段
  "generatedAt": "2026-04-24T10:32:11+08:00",  // 章节首次完成时间
  "lastUpdatedAt": "2026-04-24T11:05:42+08:00", // 末次修复时间
  "gateScores": {
    "consistency": { "score": 85, "passed": true, "issues": 1 },
    "style":       { "score": 78, "passed": true, "issues": 2 },
    "copyedit":    { "score": 82, "passed": true, "issues": 0 },
    "memory":      { "score": 90, "passed": true, "issues": 0 },
    "reader": {
      "score": 75,
      "passed": true,
      "subscores": {
        "end_hook":   85,
        "retention":  75,
        "surprise":   70,
        "immersion":  80,
        "payoff":     75,
        "share":      80
      }
    }
  },
  "gateDurationSeconds": 142,  // 门禁总耗时
  "tags": ["快档", "名场面-比武"]  // 自由标签
}
```

并在顶层增加默认阈值配置（可选）：

```jsonc
{
  // ... 原有字段 ...
  "gateThresholds": {
    "consistency": 70,
    "style": 70,
    "copyedit": 70,
    "memory": 70,
    "reader": 70
  },
  "readerProfile": "webnovel_veteran"
}
```

## 命令行参数

```bash
python3 scripts/quality_dashboard.py [参数]

  --project-root <path>          项目目录（必填）
  --range <start>-<end>          章节范围（默认所有）
  --window <N>                   趋势窗口（默认 30 章）
  --dimension <name>             仅显示指定维度（all/consistency/style/copyedit/memory/reader）
  --format <fmt>                 输出格式 cli/json/md/csv（默认 cli）
  --output <path>                输出到文件（默认 stdout）
  --threshold-warn <delta>       拐点告警阈值（默认 5 分）
  --no-color                     禁用 ANSI 颜色（CI 环境）
```

## 拐点检测算法

**目标**：识别质量出现**结构性下降**的章节。

**算法**：
1. 对每个维度的分数序列计算 5 章滑动窗口均值
2. 当 `当前 5 章均值 < 前 5 章均值 - threshold_warn` → 标记拐点
3. 同时满足"连续 ≥ 3 章低于全书均值" → 升级为告警

**优先级**：
- 🔴 严重：连续下降 ≥ 10 分 + 持续 5 章
- 🟡 警告：单次下降 ≥ 5 分 或 连续下降 ≥ 3 章
- 🟢 提示：仅单章异常

---

## 离群章 / 高光章计算

**离群章**（worst N）：
```
排序键 = (gateScores.总分均值, -最低维度分数)
取最低 N 章
```

**高光章**（best N）：
```
排序键 = (gateScores.总分均值, 最高维度分数)
取最高 N 章
```

默认 N = 5，可通过 `--top N` 调整。

## 与读者维度的联动

`reader-simulator-spec.md` 的 6 子分被特殊渲染：
- 任一子分连续 3 章 < 65 → 仪表盘单独报警
- 6 子分总和最低的章 → 标"读者险章"

## 实现要点（脚本侧）

### 输入鲁棒性

- 老 `02-写作计划.json` 缺 `gateScores` 字段时：标"无数据"，不报错
- 部分章节缺数据时：折线图用虚线连接已知点
- `gateScores.X.score` 缺失时按 0 处理但单独标注

### 性能要求

- 50 章 < 100ms
- 500 章 < 500ms
- 不依赖外部 API（纯本地计算）

### 依赖

- 仅 Python 标准库（不引入 pandas/matplotlib，保持 zero-dep）
- ASCII 图自实现（参考 `unicodedata` 的方块字符 ▏▎▍▌▋▊▉█）
